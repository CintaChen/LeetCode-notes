# LeetCode 438. 找到字符串中所有字母异位词

## 题目描述

给定两个字符串 `s` 和 `p`，找到 `s` 中所有是 `p` 的字母异位词的子串，返回这些子串的起始索引。

字母异位词指的是：两个字符串包含的字符相同，但字符顺序可以不同。

例如：

```python
s = "cbaebabacd"
p = "abc"
```

其中 `"cba"` 和 `"bac"` 都是 `"abc"` 的字母异位词，所以返回：

```python
[0, 6]
```

------

## 解题思路：固定长度滑动窗口 + 排序比较

因为要找的是 `p` 的字母异位词，所以子串的长度一定等于 `len(p)`。

因此我们可以在字符串 `s` 中维护一个长度固定为 `len(p)` 的窗口。

核心思路是：

1. 先对字符串 `p` 排序，得到标准结果 `resort`。
2. 在 `s` 中不断截取长度为 `len(p)` 的子串。
3. 对每个子串排序。
4. 如果排序后的子串和排序后的 `p` 相同，说明这个子串是 `p` 的字母异位词。
5. 记录当前窗口左边界 `left` 的位置。
6. 每次循环结束后，窗口整体向右移动一格。

------

## 为什么排序后可以比较？

字母异位词的特点是：

> 字符种类和字符数量完全一样，只是顺序不同。

例如：

```python
"abc"
"cba"
"bac"
```

它们排序后都会变成：

```python
['a', 'b', 'c']
```

所以只要两个字符串排序后相同，就说明它们互为字母异位词。

------

## 代码实现

```python
from typing import List

class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        left = 0
        outcome = []

        # 先对 p 排序，作为比较标准
        resort = sorted(p)

        # right 从 len(p) - 1 开始，保证窗口长度等于 len(p)
        for right in range(len(p) - 1, len(s)):
            # 截取当前窗口
            part = s[left:right + 1]

            # 对当前窗口排序
            part_resort = sorted(part)

            # 如果排序结果相同，说明当前窗口是 p 的字母异位词
            if part_resort == resort:
                outcome.append(left)

            # 窗口向右移动
            left += 1

        return outcome
```

------

## 示例分析

以：

```python
s = "cbaebabacd"
p = "abc"
```

为例。

`p` 排序后是：

```python
['a', 'b', 'c']
```

窗口长度固定为 `3`。

遍历过程：

```text
窗口 "cba" -> 排序后 ['a', 'b', 'c']，匹配，记录索引 0
窗口 "bae" -> 排序后 ['a', 'b', 'e']，不匹配
窗口 "aeb" -> 排序后 ['a', 'b', 'e']，不匹配
窗口 "eba" -> 排序后 ['a', 'b', 'e']，不匹配
窗口 "bab" -> 排序后 ['a', 'b', 'b']，不匹配
窗口 "aba" -> 排序后 ['a', 'a', 'b']，不匹配
窗口 "bac" -> 排序后 ['a', 'b', 'c']，匹配，记录索引 6
窗口 "acd" -> 排序后 ['a', 'c', 'd']，不匹配
```

最终返回：

```python
[0, 6]
```

------

## 关键点总结

这道题的窗口长度是固定的，始终等于 `len(p)`。

代码中的：

```python
part = s[left:right + 1]
```

表示截取当前窗口。

代码中的：

```python
if part_resort == resort:
    outcome.append(left)
```

表示如果当前窗口排序后和 `p` 排序后相同，就记录当前窗口的起始位置。

------

## 复杂度分析

设 `n` 是字符串 `s` 的长度，`m` 是字符串 `p` 的长度。

时间复杂度：

```text
O(n * m log m)
```

因为每个窗口都需要排序，而每个窗口长度是 `m`。

空间复杂度：

```text
O(m)
```

因为每次需要存储当前窗口排序后的结果。

------

## 备注

这个方法思路直观，容易理解，适合作为第一版解法。

不过由于每个窗口都要排序，所以效率不是最优。后续可以使用字符频率统计来优化，把时间复杂度降低到接近 `O(n)`。
